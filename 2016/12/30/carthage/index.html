Carthage

发表于 2016-12-30   |   分类于 学无止境   |  

iOS开发已经习惯了使用CocoaPods对第三方库进行管理，最近接触了一个新的Cocoa依赖管理工具Carthage，下面就大致介绍一下我使用的一个过程。两者区别自己看，嚯嚯~

安装

打开终端，输入一哈指令

$ brew update

$ brew install carthage

安装好以后输入$ carthage version查看当前的版本号，我当前的版本号是0.18.1。

使用

通过终端打开项目所在路径

$ cd /Path/Project

创建一个空的Carthage文件

$ touch Cartfile

打开Carthage文件

$ open -a Xcode Cartfile

在文件里输入以下内容

github "AFNetworking/AFNetworking"
github "SnapKit/Masonry"
github "rs/SDWebImage"
github "Mantle/Mantle"

保存Carthage文件，并执行一下命令

$ carthage update --platform iOS

一一一一一一一一一一500年后一一一一一一一一一一
项目的文件夹中从此有了一个名为Carthage的文件夹,然后一层一层的剥开它的心，在 Project/Carthage/Build/iOS里会出现许多不可描述的文件，但是我还是一眼在人群中找到了xxx.framework文件 =。=
然后回到了远古时代添加framework的方法，打开项目，点击project，选择target, 再选择上方的General，将需要的framework文件拖到 Linked frameworks and Binaries内
再然后点击Build Phrase tab选项，添加相应的run script，shell下面输入

/usr/local/bin/carthage copy-frameworks

input Files输入

$(SRCROOT)/Carthage/Build/iOS/AFNetworking.framework
$(SRCROOT)/Carthage/Build/iOS/Mantle.framework
$(SRCROOT)/Carthage/Build/iOS/WebImage.framework
$(SRCROOT)/Carthage/Build/iOS/Masonry.framework

好了，这就完了。

再再然后，你需要#import <AFNetworking/AFNetworking.h>来引用它。

完

嚯嚯，这下真的完了。阿鸡毛，你学到了吗
